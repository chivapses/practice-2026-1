# Документация по проектной (учебной) практике

Раздел описывает процесс прохождения проектной (учебной) практики. Практика выполняется индивидуально на основе проекта по дисциплине «Проектная деятельность» — Telegram-бота оповещений о расписании пар Московского Политеха.

- [Этапы практики](#этапы-практики)
- [Подробный журнал прогресса](practice_documentation.md)

## Описание проекта

В основе практики — Telegram-бот, разработанный ранее в рамках дисциплины «Проектная деятельность». Бот получает расписание с сервиса `rasp.dmami.ru`, позволяет указать группу и присылает напоминания перед началом пар. Подробное описание проекта — на сайте практики (раздел `site/`).

В рамках практики проект используется как основа: вокруг готового кода организуются репозиторий, документация, сайт и отчётность.

## Этапы практики

| Этап | Содержание | Статус |
|------|-----------|--------|
| 1. Настройка Git и репозитория | Репозиторий из шаблона, базовые команды, `.gitignore`, ветки | Выполнено |
| 2. Документация в Markdown | Оформление документации по практике | Выполнено |
| 3. Статический веб-сайт | Сайт о проекте: главная, о проекте, участники, журнал, ресурсы | Запланировано |
| 4. Отчёт по практике | Итоговый отчёт по шаблону в форматах DOCX и PDF | Запланировано |

Подробности по каждому этапу — в файле [practice_documentation.md](practice_documentation.md).
